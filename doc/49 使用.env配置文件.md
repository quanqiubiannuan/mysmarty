在框架的根目录下，有个`.env.example`示例文件，如果你打算用它，你需要将它重命名为 `.env` 文件。

一般来说，只需要配置数据的相关更改即可。

```
#是否开启调试模式
APP_DEBUG=true
#数据库配置
#连接主机
DB_HOST=127.0.0.1
#数据库端口
DB_PORT=3306
#数据库默认库
DB_DATABASE=test
#数据库用户名
DB_USERNAME=root
#数据库密码
DB_PASSWORD="232323"
```

因为在真正的数据库配置文件中，读取的其实是这里的数据。

```php
'mysql' => [
    // 主机ip
    'host' => env('DB_HOST', '127.0.0.1'),
    // mysql 用户名
    'user' => env('DB_USERNAME', 'root'),
    // mysql 密码
    'password' => env('DB_PASSWORD', '123456'),
    // mysql 端口
    'port' => env('DB_PORT', 3306),
    // mysql 默认数据库
    'database' => env('DB_DATABASE', 'test'),
    // mysql 字符编码
    'charset' => env('DB_CHARSET', 'utf8mb4')
],
```

你完全可以不用.env配置，env方法有2个参数，如果没有读取到这个env配置数据，则会取第二个参数的值作为默认值。

.env配置也可以配置其它数据

比如：

```
XXX_YYY="232323"
```

我们推荐使用大学字母+下划线命名。

你可以在控制器里通过 `env('XXX_YYY')` 读取到这个配置。

为了保证你的配置安全，我们建议在开发环境中用.env配置，在线上环境则将.env文件删除，统一使用php文件的配置方案（使用默认值的形式）。

例如：

```php
return [
    // cookie 保存时间
    'expire' => env('COOKIE_EXPIRE', 0),
    // cookie 保存路径
    'path' => env('COOKIE_PATH', '/'),
    // cookie 有效域名
    'domain' => env('COOKIE_DOMAIN', ''),
    //  cookie 启用安全传输
    'secure' => env('COOKIE_SECURE', false),
    // httponly设置
    'httponly' => env('COOKIE_HTTPONLY', false),
    // 是否使用 setcookie
    'setcookie' => env('COOKIE_SETCOOKIE', true)
];
```
